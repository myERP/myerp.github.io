JSON Format

Vendor bills are represented as JSON objects which have the following keys:

{% include table-header.html %}
id {{ site.format.type.integer }} no yes Automatically assigned when creating customers
locked {{ site.format.type.boolean }} no no Indicates that the document can no longer be edited nor deleted.
sent {{ site.format.type.boolean }} no no Has the document been finalized or sent to the Customer.
sent_at {{ site.format.type.date }} no no The date the document has been finalized or sent to the Customer.
Format: {{ site.format.date }}
vendor_id DEPRECATED {{ site.format.type.integer }} yes no Id of the Vendor.
vendor {{ site.format.type.snippet }} yes no The Snippet of the bill Vendor.
Where :
- id = vendor.id
- name = vendor.full_name
- user_code = vendor.user_code
billing_contact_id {{ site.format.type.integer }} no no Id of the Contact to bill.
Should belong to that bill's Vendor.
billing_address {{ site.format.type.address }} no yes The billing address. The Contact address if any, the Vendor one otherwise.
paid {{ site.format.type.double }} no yes Total of the payments made on that bill.
unpaid {{ site.format.type.double }} no yes Remaining amount to be paid.
user_code {{ site.format.type.string }} no no The number assigned to this bill (i.e. VINV-002). If you don't specify a bill number, it will be automatically generated. 64 characters max.
date {{ site.format.type.date }} no no The date that will be printed.
Format: {{ site.format.date }}
Default: date of the day.
currency_iso_code {{ site.format.type.string }} no no The ISO Code of one of your Currencies
by default, the domain currency will be selected. 4 characters max.
employee_id {{ site.format.type.integer }} no no Id of the Employee assigned to that bill.
Default: the authenticated user.
order_lines {{ site.format.type.order_line }} no no An array containing the order lines of the bill.
shipping_contact_id {{ site.format.type.integer }} no no Id of the Contact the goods should be shipped to, if different than the Vendor.
Could be any Contact from any Vendor. By default the user's address.
shipping_address {{ site.format.type.address }} no yes The shipping address. The shipping Contact address if any, the User's one otherwise.
payment_terms_id {{ site.format.type.integer }} no no Id of the Payment Term to apply on that bill. Will be used as a template to generate Settlements.
private_note {{ site.format.type.string }} no no A private note that won't be printed.
subtotal {{ site.format.type.double }} no yes Total before taxes and reduce. Based on the content of the order lines.
reduce_amount {{ site.format.type.double }} no yes Amount of the reduce, based on the subtotal and reduce rate.
tax_amount {{ site.format.type.double }} no yes Overall tax amount. Includes all taxes, discounts and reduce.
total {{ site.format.type.double }} no yes The final total the user needs to pay to the vendor. All include (taxes, discounts, reduce).
